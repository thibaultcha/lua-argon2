Module argon2

Lua C binding for the Argon2 password hashing algorithm.

See the Argon2 documentation at the same time while you consult this binding's documentation.

Note: this document is also valid for [lua-argon2-ffi] (https://github.com/thibaultcha/lua-argon2-ffi) which uses the same prototype for its encrypt and verify functions.

Functions

encrypt (plain, salt, options)
Encrypt a plain string. Use Argon2i (by default) or Argon2d to hash a string.

Parameters:

  • plain string Plain string to encrypt.
  • salt string Salt to use to hash pwd.
  • options table Options with which to hash the plain string. See options. This parameter is optional, if values are omitted the default ones will be used.

Returns:

  1. string hash: Hash computed by Argon2 or nil if an error occurred.
  2. string error: nil or a string describing the error if any.

Usage:

    local hash, err = argon2.encrypt("password", "somesalt")
local hash, err = argon2.encrypt("password", "somesalt", {t_cost = 4})
verify (encrypted, plain)
Verify a plain string against a hash. Uses Argon2i or Argon2d to verify if a given password (or any plain string) verifies against a hash encrypted with Argon2.

Parameters:

  • encrypted string Hash to verify the plain string against.
  • plain string Plain string to verify.

Returns:

  1. boolean ok: true if the password matched, false otherwise.
  2. string error: nil or a string describing the error if any.

Usage:

    local ok, err = argon2.verify(argon2i_hash, "password")
local ok, err = argon2.verify(argon2d_hash, "password")

Tables

options
Argon2 hashing options. Those options can be given to encrypt as a table. If values are omitted, the default values will be used. Default values can be overriden with m_cost, t_cost, parallelism and argon2d.

Fields:

  • t_cost

    Number of iterations (number, default: 2). Can be set to a new default in lua-argon2 (C binding only) by calling:

    argon2.t_cost(4)
  • m_cost

    Sets memory usage to m_cost kibibytes (number, default: 12). Can be set to a new default in lua-argon2 (C binding only) by calling:

    argon2.m_cost(4)
  • parallelism

    Number of threads and compute lanes (number, default: 1). Can be set to a new default in lua-argon2 (C binding only) by calling:

    argon2.parallelism(2)
  • argon2d

    If true, will use the Argon2d hashing function (boolean, default: false). Can be set to a new default in lua-argon2 (C binding only) by calling:

    argon2.argon2d(true)
argon2.argon2d("on")
